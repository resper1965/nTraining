## Style guide do conteúdo — n.secops (UX/UI aplicado ao material)

Este guia aplica os preceitos que “ingerimos” (clareza, hierarquia, consistência e densidade) ao **conteúdo do curso** em Markdown, para ficar fácil de ler e de operar no dia a dia.

### Princípios (como UX/UI, mas para texto)
- **Direção**: material “enterprise” (preciso, denso, sem floreio).
- **Hierarquia clara**: títulos curtos, seções previsíveis, bullets objetivos.
- **Progressive disclosure**: primeiro o mínimo viável (o que fazer), depois detalhes (por quê).
- **Consistência**: todos os módulos usam o mesmo formato (README → aulas → quiz).
- **Acionável**: sempre terminar com checklist/exercício/output esperado.

### Padrão de cada módulo
Cada módulo deve ter:
- `README.md` com: objetivo, duração, resultados, pré-requisitos, aulas, prática, avaliação.
- `lessons/` com 3 aulas (salvo exceções).
- `quiz.md` (Parte 1) ou `lab.md` (Parte 2), quando fizer mais sentido.

### Padrão de cada aula
Cada aula deve ter:
- **Objetivo**
- **Conceitos‑chave**
- **Checklist operacional** (se aplicável)
- **Exercício** + **saída esperada**
